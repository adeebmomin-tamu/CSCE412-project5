TIC Chatbot

This project was developed as a result of my machine learning research. In my research, we fine tuned a model based off of Google Gemini with lectures provided by the supervising professor. These lectures focused on industry control, with things like programmable logic controllers and IO relays. Using this data, we fine tuned the model by providing Gemini data through Google AI Studio. From there, we created the frontend for a chatbot that would be used in classrooms for further testing. Using React, CSS, JavaScript, Node.js, and Python, we were able to develop the whole chatbot with responsive web design and quick responses. Firebase was utilized as well to store data entries and chat logs, along with user information. This project was a great learning experience, as I was able to get hands on experience using API calls, developing using React, working with fine tuning data, and even creating my own chatbot. This project is still being worked on as there are many aspects that can still be improved, like making the fine tuned model faster than it currently is.

Satisfaction Scanner

This project was developed during the TAMU HowdyHack hackathon is September 2023. In just 24 hours, my team and I were able to create Satisfaction Scanner using Python for backend, PyQT5 for frontend, Pandas for data management, and OpenCV for the facial recognition software. The project utilizes OpenCV to scan and detect human emotions. From there, it categorizes them into specific emotions, like satisfied, unsatisfied, and sad. From there, this data was stored in an excel file and sent to PowerBI for data visualization. For the frontend, we utilized PyQT5 as we were still novices at frontend development. This project was a valuable learning experience for me, as collaborating with 2 others gave me experience coding within a group. Using Git was one aspect that I really took away from this hackathon.
